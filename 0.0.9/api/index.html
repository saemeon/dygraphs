    API Reference¶

    Core¶

    Builder for a dygraphs chart — framework-agnostic config object.

    Port of the R dygraph() constructor and the full family of dy*() pipe functions (dyOptions, dySeries, dyAxis …), expressed as chained method calls. Produces a plain dict via :meth:to_dict / :meth:to_js, standalone HTML via :meth:to_html, or plugs into :class:dygraphs.dash.DygraphChart (Dash) and :func:dygraphs.shiny.dygraph_ui / :func:~dygraphs.shiny.render_dygraph (Shiny) for interactive rendering.

    Two equivalent construction styles are supported and can be mixed:

    • Builder (chained methods) — mirrors R's pipe idiom, the most common style. All configuration happens via methods like :meth:options, :meth:series, :meth:axis, :meth:legend after construction.
    • Declarative (keyword dataclasses) — the same configuration can be passed to Dygraph(...) directly via the declarative parameters below. Each accepts either a dataclass instance (Options(fill_graph=True)) or a plain dict with the same keys ({"fill_graph": True}). Useful when loading config from YAML / JSON, or when users prefer a single constructor call over method chaining.
    PARAMETER DESCRIPTION
    data

    Chart data. Accepted shapes:

    • pd.DataFrame — index is the x-axis (DatetimeIndex → date axis, otherwise numeric); columns are the series.
    • pd.Series — promoted to a single-column DataFrame.
    • dict — first key is x-axis, remaining keys are series.
    • list[list] — row-oriented; first column is x-axis.
    • np.ndarray — 1-D treated as a single series, 2-D as rows.
    • str — parsed as CSV. First column becomes the x-axis.

    Mixed timezone / naive datetime indexes are normalised to ISO strings on output.

    TYPE: pd.DataFrame | pd.Series | dict[str, list] | list[list] | np.ndarray | str

    title

    Chart title (main in R). Rendered above the plot. By default None.

    TYPE: str | None DEFAULT: None

    xlab

    X-axis label. By default None (inferred from the data: index name if named, else "Date" for date axes / "x" otherwise).

    TYPE: str | None DEFAULT: None

    ylab

    Y-axis label. By default None.

    TYPE: str | None DEFAULT: None

    periodicity

    Manually override the auto-detected periodicity of date data. Only valid for date-formatted data — passing it with numeric data raises :class:ValueError. Mirrors R's dygraph(..., periodicity=...). By default None, in which case the scale is inferred from the pandas index spacing.

    TYPE: ('yearly', 'quarterly', 'monthly', 'weekly', 'daily', 'hourly', 'minute', 'seconds', 'milliseconds') DEFAULT: "yearly"

    group

    Cross-chart synchronisation group name. Charts that share the same name auto-sync their x-axis zoom, pan, and highlight. Mirrors R's dygraph(group=). Equivalent to the chainable :meth:sync_group alias. Not the same as the :meth:group builder method, which takes a list of column names and mirrors R's dyGroup() to style a set of series together — the name collision is intentional, both mirror the R API verbatim. By default None (no group — chart is standalone).

    TYPE: str | None DEFAULT: None

    width

    Chart width in pixels. By default None (auto — fills container).

    TYPE: int | None DEFAULT: None

    height

    Chart height in pixels. By default None (auto).

    TYPE: int | None DEFAULT: None

    PARAMETER DESCRIPTION
    options

    Global dygraph options (dyOptions in R). Field names match the builder method :meth:options — e.g. Options(fill_graph=True, stroke_width=2) or {"fill_graph": True, "stroke_width": 2}. Sets things like stacking, fill, stroke style, colour palette, rolling period, etc. Full field list: :class:dygraphs.declarative.Options. By default None.

    TYPE: :class:`~dygraphs.Options` | dict | None

    axes

    Per-axis configuration (dyAxis in R). Accepts a dict keyed by axis name ("x", "y", "y2") or a list of :class:~dygraphs.Axis / dicts (each must include name). Configures label text, value range, tick formatting, log scale, grid, etc. Full field list: :class:dygraphs.declarative.Axis. By default None.

    TYPE: dict[str, :class:`~dygraphs.Axis`] | list[:class:`~dygraphs.Axis`] | None

    series

    Per-series display options (dySeries in R). Each entry configures one series by name — colour, stroke width, axis assignment, step plot, etc. Full field list: :class:dygraphs.declarative.Series. By default None.

    TYPE: list[:class:`~dygraphs.Series` | dict] | None

    legend

    Legend configuration (dyLegend in R): show mode ("auto" / "always" / "follow" / "never"), width, label layout, follow-cursor offsets. Full field list: :class:dygraphs.declarative.Legend. By default None.

    TYPE: :class:`~dygraphs.Legend` | dict | None

    highlight

    Hover-highlight behaviour (dyHighlight in R): circle size, highlight-series-opts, hide-on-mouseout. By default None.

    TYPE: :class:`~dygraphs.Highlight` | dict | None

    annotations

    List of annotations (dyAnnotation in R). Each points at an x-value with short text, tooltip, colour, and optional click handlers. By default None.

    TYPE: list[:class:`~dygraphs.Annotation` | dict] | None

    shadings

    List of shaded regions (dyShading in R). Each has from_ and to bounds on the x- or y-axis, plus a fill colour. By default None.

    TYPE: list[:class:`~dygraphs.Shading` | dict] | None

    events

    List of vertical event lines (dyEvent in R). Each pins a labelled line at an x-value. By default None.

    TYPE: list[:class:`~dygraphs.Event` | dict] | None

    limits

    List of horizontal limit lines (dyLimit in R). By default None.

    TYPE: list[:class:`~dygraphs.Limit` | dict] | None

    range_selector

    Range-selector configuration (dyRangeSelector in R): height, initial date_window, fill/stroke colours, keep_mouse_zoom, retain_date_window. See :class:dygraphs.declarative.RangeSelector. By default None (no range selector).

    TYPE: :class:`~dygraphs.RangeSelector` | dict | None

    roller

    Rolling-average widget (dyRoller in R): show widget, initial roll period. By default None.

    TYPE: :class:`~dygraphs.Roller` | dict | None

    callbacks

    JavaScript event callbacks (dyCallbacks in R): click, zoom, highlight, draw, point-click handlers. Values are raw JS source strings (wrapped in :class:~dygraphs.JS for type clarity, but plain strings also work). By default None.

    TYPE: :class:`~dygraphs.Callbacks` | dict | None

    plotter

    Chart-level plotter shortcut — one of "bar_chart", "stacked_bar_chart", "multi_column", "candlestick". Equivalent to calling the matching builder method (e.g. :meth:bar_chart) after construction. By default None (default line plotter).

    TYPE: str | None

    RAISES DESCRIPTION
    ValueError

    If periodicity is given for numeric (non-date) data, if periodicity is not one of the accepted values, or if data has an unsupported shape (e.g. empty CSV, < 2 columns).

    Examples:

    From a pandas DataFrame (builder style):

    >>> import pandas as pd
>>> df = pd.DataFrame(
...     {"temp": [10, 12, 11], "rain": [5, 3, 7]},
...     index=pd.date_range("2024-01-01", periods=3),
... )
>>> chart = (
...     Dygraph(df, title="Weather")
...     .options(fill_graph=True, stroke_width=2)
...     .series("temp", color="red")
...     .axis("y", label="°C")
...     .range_selector(height=30)
... )

    The same chart in declarative form:

    >>> from dygraphs import Options, Series, Axis, RangeSelector
>>> chart = Dygraph(
...     df,
...     title="Weather",
...     options=Options(fill_graph=True, stroke_width=2),
...     series=[Series("temp", color="red")],
...     axes={"y": Axis(name="y", label="°C")},
...     range_selector=RangeSelector(height=30),
... )

    Declarative accepts plain dicts too — mix freely:

    >>> chart = Dygraph(
...     df,
...     options={"fill_graph": True},
...     series=[Series("temp", color="red"), {"name": "rain", "axis": "y2"}],
... )

    From a CSV string:

    >>> chart = Dygraph("Date,A,B\n2024-01-01,1,2\n2024-01-02,3,4")

    Synchronise two charts on the x-axis:

    >>> a = Dygraph(df1, group="weather").range_selector()
>>> b = Dygraph(df2, group="weather").range_selector()

    Render to different targets:

    >>> html = chart.to_html()        # standalone HTML page
>>> config = chart.to_dict()      # raw Python dict (introspection)
>>> payload = chart.to_js()       # JSON-safe dict (Dash / Shiny)
    Notes

    Dygraph is intentionally framework-agnostic — same pattern as :class:plotly.graph_objects.Figure and R's htmlwidgets. To render in Dash, wrap with :class:dygraphs.dash.DygraphChart. To render in Shiny, use :func:dygraphs.shiny.dygraph_ui in the UI and :func:dygraphs.shiny.render_dygraph in the server. For standalone rendering (reports, emails, iframes), :meth:to_html returns a complete HTML page.

    See Also

    options : Set global dygraph options (dyOptions). series : Configure a single series by name (dySeries). axis : Configure an axis (dyAxis). legend : Configure the legend (dyLegend). highlight : Configure hover highlighting (dyHighlight). annotation : Add a single annotation (dyAnnotation). shading : Add a shaded region (dyShading). event : Add a vertical event line (dyEvent). limit : Add a horizontal limit line (dyLimit). range_selector : Add the bottom range-selector widget. roller : Add the rolling-average widget. callbacks : Attach JS event callbacks (dyCallbacks). sync_group : Chainable alias for the constructor group= kwarg. group : Style a list of series together (dyGroup) — do not confuse with the group= constructor kwarg. to_dict : Serialise to a Python dict (raw JS objects retained). to_js : Serialise to a JSON-safe dict with __JS__:code:__JS__ markers (for Dash / Shiny callback returns). to_html : Render to a self-contained HTML page. dygraphs.dash.DygraphChart : Dash component wrapper. dygraphs.shiny.dygraph_ui : Shiny UI helper. dygraphs.shiny.render_dygraph : Shiny server-side render helper.

    Set global chart options.

    Mirrors R dyOptions.

    PARAMETER DESCRIPTION
    stacked_graph

    If set, stack series on top of one another rather than drawing them independently. By default False.

    TYPE: bool DEFAULT: False

    fill_graph

    Fill the area underneath the graph. Not compatible with error bars. Can also be set per-series. By default False.

    TYPE: bool DEFAULT: False

    fill_alpha

    Transparency for filled regions (0.0 = invisible, 1.0 = as dark as the line). By default 0.15.

    TYPE: float DEFAULT: 0.15

    step_plot

    Display as a step plot instead of a line plot. Can also be set per-series. By default False.

    TYPE: bool DEFAULT: False

    stem_plot

    Display as a stem plot instead of a line plot. Can also be set per-series. By default False.

    TYPE: bool DEFAULT: False

    draw_points

    Draw a small dot at each data point. Can also be set per-series. By default False.

    TYPE: bool DEFAULT: False

    point_size

    Size in pixels of the dot drawn at each point. Also controls the size of dots at isolated points. By default 1.0.

    TYPE: float DEFAULT: 1.0

    point_shape

    Shape of the dot drawn at each point. One of "dot", "triangle", "square", "diamond", "pentagon", "hexagon", "circle", "star", "plus", or "ex". By default "dot".

    TYPE: str DEFAULT: 'dot'

    draw_gap_edge_points

    Draw points at the edges of gaps in the data. By default False.

    TYPE: bool DEFAULT: False

    connect_separated_points

    If True, connect points even when there are missing values in between (useful when different series have data at different x-values). By default False.

    TYPE: bool DEFAULT: False

    stroke_width

    Width in pixels of the lines connecting data points. Can also be set per-series. By default 1.0.

    TYPE: float DEFAULT: 1.0

    stroke_pattern

    A predefined pattern ("dotted", "dashed", "dotdash", "solid") or a custom array where even indices are draw lengths and odd are space lengths in pixels. Can also be set per-series. By default None (solid line).

    TYPE: str | list[int] | None DEFAULT: None

    stroke_border_width

    Draw a border around graph lines to make crossing lines more distinguishable. Can also be set per-series. By default None.

    TYPE: float | None DEFAULT: None

    stroke_border_color

    Color for the line border when stroke_border_width is set. Can also be set per-series. By default "white".

    TYPE: str DEFAULT: 'white'

    plotter

    A JavaScript function (or array of functions) which plots each data series. May also be set per-series. By default None.

    TYPE: str | None DEFAULT: None

    colors

    Colors for the data series, e.g. "#AABBCC", "rgb(255,100,200)", or "yellow". If not specified, equally-spaced points around a colour wheel are used. By default None.

    TYPE: list[str] | None DEFAULT: None

    color_value

    Value component (0.0--1.0) of auto-generated series colours in HSV space. By default 0.5.

    TYPE: float DEFAULT: 0.5

    color_saturation

    Saturation component (0.0--1.0) of auto-generated series colours in HSV space. By default 1.0.

    TYPE: float DEFAULT: 1.0

    draw_x_axis

    Whether to draw the x-axis. Setting to False also prevents x-axis ticks and reclaims the space. By default True.

    TYPE: bool DEFAULT: True

    draw_y_axis

    Whether to draw the y-axis. Setting to False also prevents y-axis ticks and reclaims the space. By default True.

    TYPE: bool DEFAULT: True

    include_zero

    Always include zero in the y-axis range, typically as the lowest value. By default False.

    TYPE: bool DEFAULT: False

    draw_axes_at_zero

    Draw the X axis at Y=0 and the Y axis at X=0 if those positions are inside the visible area; otherwise draw at the edges as usual. By default False.

    TYPE: bool DEFAULT: False

    logscale

    Show the y-axis in log scale. Values <= 0 are not displayed. By default False.

    TYPE: bool DEFAULT: False

    axis_tick_size

    Spacing between axis labels and tick marks. By default 3.0.

    TYPE: float DEFAULT: 3.0

    axis_line_color

    Color of x- and y-axis lines (CSS string). By default "black".

    TYPE: str DEFAULT: 'black'

    axis_line_width

    Thickness in pixels of x- and y-axis lines. By default 0.3.

    TYPE: float DEFAULT: 0.3

    axis_label_color

    Color for x- and y-axis labels (CSS string). By default "black".

    TYPE: str DEFAULT: 'black'

    axis_label_font_size

    Font size in pixels for axis labels. By default 14.

    TYPE: int DEFAULT: 14

    axis_label_width

    Width in pixels of the divs containing axis labels. By default 60.

    TYPE: int DEFAULT: 60

    draw_grid

    Whether to display grid lines. Can also be set per-axis. By default True.

    TYPE: bool DEFAULT: True

    grid_line_color

    Color of grid lines. By default None.

    TYPE: str | None DEFAULT: None

    grid_line_width

    Thickness in pixels of grid lines. By default 0.3.

    TYPE: float DEFAULT: 0.3

    title_height

    Height in pixels of the chart title; also controls its default font size. By default None.

    TYPE: int | None DEFAULT: None

    right_gap

    Pixels of blank space at the right edge, making it easier to highlight the rightmost point. By default 5.

    TYPE: int DEFAULT: 5

    digits_after_decimal

    Number of digits after the decimal point in normal display mode. By default 2.

    TYPE: int DEFAULT: 2

    labels_kmb

    Show K/M/B for thousands/millions/billions on y-axis. By default False.

    TYPE: bool DEFAULT: False

    labels_kmg2

    Show k/M/G for kilo/Mega/Giga on y-axis (base 2). By default False.

    TYPE: bool DEFAULT: False

    labels_utc

    Show date/time labels in UTC instead of local time. Incompatible with use_data_timezone. By default False.

    TYPE: bool DEFAULT: False

    max_number_width

    If a number has more than this many digits left of the decimal, switch to scientific notation. By default 6.

    TYPE: int DEFAULT: 6

    sig_figs

    Display a fixed number of significant figures instead of fixed decimal places. By default None.

    TYPE: int | None DEFAULT: None

    pan_edge_fraction

    Farthest a graph may be panned, as a fraction of the display (e.g. 0.1 = 10%). None means no bounds. By default None.

    TYPE: float | None DEFAULT: None

    animated_zooms

    Animate the transition between zoom windows. By default False.

    TYPE: bool DEFAULT: False

    disable_zoom

    Disable click-and-drag zooming. By default False.

    TYPE: bool DEFAULT: False

    retain_date_window

    Keep the current zoom level when updating the chart with new data. By default False.

    TYPE: bool DEFAULT: False

    error_bars

    Enable symmetric error bars. By default False.

    TYPE: bool DEFAULT: False

    custom_bars

    Enable custom (asymmetric) bars. By default False.

    TYPE: bool DEFAULT: False

    sigma

    When error_bars is True, number of standard deviations to shade. By default None.

    TYPE: float | None DEFAULT: None

    fractions

    If True, values are interpreted as fractions and error bars are drawn using a Wilson confidence interval. By default False.

    TYPE: bool DEFAULT: False

    wilson_interval

    Use a Wilson confidence interval when fractions is True. By default True.

    TYPE: bool DEFAULT: True

    visibility

    Array of booleans controlling per-series visibility. By default None.

    TYPE: list[bool] | None DEFAULT: None

    legend_formatter

    JavaScript function to format legend values. By default None.

    TYPE: str | None DEFAULT: None

    range_selector_plot_line_width

    Line width in the range selector mini plot. By default None.

    TYPE: float | None DEFAULT: None

    range_selector_plot_fill_gradient_color

    Gradient color in the range selector fill. By default None.

    TYPE: str | None DEFAULT: None

    range_selector_background_line_width

    Background line width in the range selector. By default None.

    TYPE: float | None DEFAULT: None

    range_selector_background_stroke_color

    Background stroke color in the range selector. By default None.

    TYPE: str | None DEFAULT: None

    range_selector_foreground_stroke_color

    Foreground stroke color in the range selector. By default None.

    TYPE: str | None DEFAULT: None

    range_selector_foreground_line_width

    Foreground line width in the range selector. By default None.

    TYPE: float | None DEFAULT: None

    range_selector_alpha

    Alpha transparency of the range selector. By default None.

    TYPE: float | None DEFAULT: None

    grid_line_pattern

    Custom dash pattern for grid lines. By default None.

    TYPE: list[int] | None DEFAULT: None

    resizable

    If set (e.g. "both"), allow the chart to be resized by the user. By default None.

    TYPE: str | None DEFAULT: None

    pixel_ratio

    Custom pixel ratio for high-DPI rendering. By default None.

    TYPE: float | None DEFAULT: None

    stacked_graph_nan_fill

    How to handle NaN in stacked graphs. By default None.

    TYPE: ('all', 'inside', 'none') DEFAULT: "all"

    animate_background_fade

    Animate the background fade effect. By default True.

    TYPE: bool DEFAULT: True

    x_label_height

    Height in pixels of the x-axis label. By default None.

    TYPE: int | None DEFAULT: None

    y_label_width

    Width in pixels of the y-axis label. By default None.

    TYPE: int | None DEFAULT: None

    legend_follow_offset_x

    Horizontal pixel offset for the follow legend. By default None.

    TYPE: int | None DEFAULT: None

    legend_follow_offset_y

    Vertical pixel offset for the follow legend. By default None.

    TYPE: int | None DEFAULT: None

    range_selector_veil_colour

    Color of the veil drawn over deselected areas of the range selector. By default None.

    TYPE: str | None DEFAULT: None

    delimiter

    Delimiter for CSV data parsing. By default None.

    TYPE: str | None DEFAULT: None

    x_value_parser

    JavaScript function to parse x-values from CSV. By default None.

    TYPE: str | None DEFAULT: None

    display_annotations

    Display annotations from data columns. By default False.

    TYPE: bool DEFAULT: False

    data_handler

    Custom JavaScript data handler. By default None.

    TYPE: str | None DEFAULT: None

    mobile_disable_y_touch

    Disable touch events on the Y axis for mobile devices to avoid interfering with scrolling. By default True.

    TYPE: bool DEFAULT: True

    use_data_timezone

    Use the timezone of the underlying data for display. Incompatible with labels_utc. By default False.

    TYPE: bool DEFAULT: False

    RETURNS DESCRIPTION
    Dygraph

    Self, for chaining.

    Examples:

    >>> chart = Dygraph(df).options(
...     fill_graph=True,
...     draw_points=True,
...     colors=["#00d4aa", "#f4a261"],
...     stroke_width=2,
... )

    >>> chart = Dygraph(df).options(
...     stacked_graph=True,
...     grid_line_color="#ddd",
...     axis_label_font_size=12,
... )

    Configure a specific axis.

    Mirrors R dyAxis. Options that are not set explicitly use the global default from :meth:options.

    PARAMETER DESCRIPTION
    name

    Axis to configure.

    TYPE: ('x', 'y', 'y2') DEFAULT: "x"

    label

    Label to display for the axis. By default None.

    TYPE: str | None DEFAULT: None

    value_range

    Explicitly set the range to (low, high). If either limit is None it is calculated automatically. By default None.

    TYPE: tuple[float | None, float | None] | None DEFAULT: None

    logscale

    Show this axis in log scale. Values <= 0 are not displayed. Ignored for date-based x-axes. By default None.

    TYPE: bool | None DEFAULT: None

    ticker

    JavaScript function to generate custom tick marks. By default None.

    TYPE: str | None DEFAULT: None

    range_pad

    Extra space in pixels around the value range to keep edge points visible. By default None.

    Bases: ComponentWrapper

    Dash component for a dygraphs chart.

    Usage mirrors dcc.Graph::

    from dygraphs import Dygraph
from dygraphs.dash import DygraphChart

app.layout = html.Div([
    DygraphChart(figure=dg, id="chart"),
])

    What it actually is. A DygraphChart is an html.Div that contains two siblings:

    • dcc.Store(id=id, data=<serialised config>) — the chart store. The chart's identity in the callback registry. Its id equals the user-supplied id.
    • html.Div(id=f"{id}-container") — the container where dygraphs renders its JS-driven DOM.

    The class inherits from :class:dash_wrap.ComponentWrapper with the Store as the inner component, so the callback registry resolves Output(chart, ...) to the Store's id. Two small pieces of machinery make that work — both shipped by dash-wrap, both documented in its README, no magic beyond that:

    • _set_random_id is overridden on the wrapper to return the inner Store's id. Dash's dependency wiring calls this when it sees a component passed to Output / Input / State.
    • __class__ is a property that returns the inner Store's class, so isinstance(chart, dcc.Store) is True. The C-level type(chart) is still DygraphChart, which is what Dash's serialiser uses to emit the DOM (a normal <div>).

    A per-instance clientside callback is registered at construction. It listens to the store and dispatches to window.dygraphsDash.render(setup, config). The callback body inlines dash_render.js (IIFE-guarded, idempotent) and a tiny per-chart setup payload built by :func:_build_render_js.

    Any change to chart options — stroke width, colors, legend — is pushed through the same primary store as a fresh config (full destroy+recreate, matching R's renderDygraph model). There's no separate "options override" channel; one write path, one mental model. See CLAUDE.md "Decisions deferred / opts store" if you hit a performance wall and want to reintroduce a cheaper cosmetic-updates channel.

    PARAMETER DESCRIPTION
    figure

    Configured Dygraph builder instance, or None for an empty placeholder. With None the primary store holds data=None and the clientside renderer early-returns (if (!config) return;); users push the first real config via Output(chart, "data"). Use case: building an app layout where the first chart state arrives only after a callback fires (e.g. a button click).

    TYPE: Dygraph | None

    id

    User-facing DOM id. Assigned to the primary dcc.Store; the opts store gets {id}-opts and the container div {id}-container. Auto-generated if omitted.

    TYPE: str | None DEFAULT: None

    height

    Chart height in pixels or a CSS height string.

    TYPE: str | int DEFAULT: '400px'

    width

    CSS width for the container div.

    TYPE: str DEFAULT: '100%'

    modebar

    Show the Plotly-style hover overlay (capture + reset-zoom).

    TYPE: bool DEFAULT: True

    Capture¶

    Create a capture strategy compatible with dash_capture.capture_element().

    Composites every visible <canvas> inside the target element onto a single white-backed canvas and overlays the HTML layer (chart title, axis labels, tick labels, legend, annotations) via html2canvas, returning a base64 data-URI. Designed for charts produced by :class:dygraphs.dash.DygraphChart.

    This is a thin wrapper around :func:dash_capture.multi_canvas_strategy that pre-fills :data:DYGRAPH_HIDE_SELECTORS when hide_range_selector is True.

    PARAMETER DESCRIPTION
    hide_range_selector

    If True, temporarily hide the range-selector canvases (:data:DYGRAPH_HIDE_SELECTORS) before capture and restore their original display values afterwards.

    TYPE: bool DEFAULT: True

    format

    Output image format. The format is baked into the JS at strategy build time, so the wizard's runtime fmt argument is ignored — build a separate strategy per format if you need a chooser.

    TYPE: ('png', 'jpeg', 'webp') DEFAULT: "png"

    debug

    Forward to :func:multi_canvas_strategy — logs per-canvas blits and outlines destination rects in red.

    TYPE: bool DEFAULT: False

    settle_frames

    rAF ticks to await between live-resize and capture, so dygraphs's ResizeObserver (in :file:render_core.js) re-lays-out the canvases at the new aspect.

    TYPE: int DEFAULT: 2

    _params

    Internal hook mirroring :func:dash_capture.html2canvas_strategy. Pass inspect.signature(renderer).parameters to enable target-size capture: when the renderer declares capture_width / capture_height, the wrapped strategy emits a live-resize preprocess.

    TYPE: Mapping or None DEFAULT: None

    RETURNS DESCRIPTION
    CaptureStrategy

    A dash_capture.strategies.CaptureStrategy instance, ready to pass to capture_element(..., strategy=...).
    RAISES DESCRIPTION
    ImportError

    If dash-capture is not installed. Install with pip install dygraphs[dash] dash-capture.
    Notes

    SVG output is not supported: dygraphs renders to <canvas>, and canvas.toDataURL only emits raster formats.

    The HTML overlay rendered by html2canvas is rasterised at the same devicePixelRatio as the canvas composite, so text stays crisp on retina displays. Compositing order is canvases first, overlays on top — so labels never get drawn over by axis lines.

    Wire a capture button next to a chart::

    chart_component,
    capture_element(
        "sales-container",
        trigger="Download PNG",
        strategy=dygraph_strategy(hide_range_selector=True),
        filename="sales.png",
    ),
])

    The element id passed to capture_element is f"{id}-container" — the inner <div> that holds the chart canvases (see :class:dygraphs.dash.DygraphChart).
